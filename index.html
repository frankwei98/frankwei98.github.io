开户炒美股，足不出户投资科技公司

最近我发现许多朋友对美股感兴趣。于是今天聊这个。

今天的内容基本上来自于我炒美股半年之间的经验。

如果你对美股感兴趣但手头没账户，那么考验你的执行力的时候又来了。（这次难度大大降低！）

为什么投资美股

  • 投资自己熟悉的公司 大家平时接触的公司，都是在美国上市的公司。在A股，基本上买不到他们的股票。在美国的股票市场，你能买到百度、阿里巴巴、搜狐、新浪、微博（独立上市）、网易这些国内有名的互联网公司 ，也能买到巨人、畅游、完美世界这些国内游戏大厂，当然，也能买到苹果、Google、微软、IBM这样的国际IT巨人 ，如果对实体零售业感兴趣，也可以投资 麦当劳、百胜集团(肯德基)、星巴克、沃尔玛这些有名的实体零售业 公司 。而在国内，A股市场扔给你一堆OO化工，XX重工让你去感受一下。

  • 市场成熟，透明度高 相信大家总能听到A股的所谓的「内幕消息/交易」吧，在美国，SEC（证监会）打击内幕交易行为可是非常严厉的 炒股有保障

  • 允许做空，横竖都能赚钱 A股不能做空 美股可以 这样不管市场是好是坏 只要判定正确 总能赚大钱! (做空个股需要券商支持，且要开设保证金账户 购买做空的ETF不受限制)

  • 开户简单 与A股线上开户不同，美股开户一般没有视频人工验证的过程，害羞同学的福音! 参考下面的开户流程就知道真的很简单

  • T+0交易随时卖 当天买可以当天卖，新手可以及时止损

开户流程

材料准备：

  • 身份证
  • 一张白纸
  • 一支笔(签名用)
  • 银行网银账户(最好是香港的 入出金方便)

材料备齐之后，开户有3个步骤：

选择自己喜欢的券商 & 注册

目前我们有几个券商可以选

国内的有 老虎证券 (小米系) BBAE (前身积木股票，前小米系) 和 富途证券(腾讯系)

国外的更多了 SogoTrade、Firstrade、Scottrade、盈透Interactive Broker

这篇文章中，我们以开户最简单 的 老虎证券 和 BBAE 为例

为什么选择老虎证券或 BBAE ?

老虎证券
  • 可以买港股 老虎的清算商是盈透，名气大。不仅可以买美股 还可以买港股
  • 可以做空 支持开设保证金账户，做空股票不是梦! BBAE 暂时开不了保证金账户 也无法做空
  • 开户相对简单 资料真实即可通过审核 ( BBAE 需要你把收入情况做的好看一点)
  • 背景大，真·小米系 与前身为小米系积木股票现在身份成谜的 BBAE 相比，老虎证券可是一直的小米系公司，还有多个VC跟投，比较有保障
  • 名气大 很多人讨论老虎证券 很少人讨论 BBAE
  • 真·交易软件 与软件封装HTML5网页做 Windows / Mac 交易端的 BBAE 不同，老虎证券的所有交易软件都是独立编写的软件
  • 入金速度快 老虎证券用的是盈透证券，而盈透证券每一个客户都有一个专用的花旗银行入金账户，建设银行跨国转账只需半小时就能成功入金! Apex清算的BBAE、SogoTrade 、Firstrade 所用的是开设在BMO Harris Bank公共入金账户，至少1天才能到账
  • 资金安全 老虎证券客户的现金余额都是在清算商盈透合作的银行花旗银行，安全有保障
  • 准盈透账户 每个玩美股的人都有一个开户盈透证券的梦（功能强大、手续费低），而老虎证券实际上就是盈透证券的贴牌货，如果你喜欢盈透的强大工具，你可以联系客服获取你的盈透交易账户和密码
BBAE(前身积木股票)
  • 手续费业界最低 交易手续费比老虎的2.99刀还要再便宜一刀
  • 入金限制最低 官方无明确规定最低入金额，只是建议每次入金1000美元起 ，而老虎证券有首次最低入金3000美元的要求
  • 有网页端 意味着你用 Linux 系统，也可以使用浏览器来交易,而老虎证券需要客户端才能交易

再见 Ghost,你好 Hexo

为什么抛弃Ghost?

  • 插件系统安装不方便
  • Ghost博客迁移过程麻烦
  • 不能Typecho和Wordpress一样通过Web界面自定义主题的参数，导致不修改主题文件难以实现自定义
  • 有时间待在电脑前写博客了
  • 放在Github Pages上可以永久免费存放

为什么选择Hexo？

  • 简单 多中文的技术文档、主题
  • 和Ghost一样，都是使用Markdown格式
  • 有从Ghost迁移到Hexo的教程

迁移教程

首先，把Node.js和Git安装好 (各大系统安装方法不同，请参考https://hexo.io/zh-cn/docs/index.html)

安装Hexo

npm install -g hexo-cli
hexo init
npm install

修改_config.yml来满足自己的需要

导出博文

打开 http://yourghostblog.com/ghost/settings/labs/

选择导出Ghost博客设置和数据，保存为ghost.json

安装hexo-migrator-ghost

执行以下命令

npm install hexo-migrator-ghost --save
hexo migrate ghost ghost.json

这样你的Ghost博文都会被导出成.md的文件了

修复标签Bug

貌似Markdown标准不统一的原因，Ghost和Hexo对Tag的处理方式不同

所以我们需要通过替换来修复标签的bug

我用的是Atom编辑器，对整个文件夹的markdown文件进行了替换

只需要把 tags: | 替换成 tags:

保持博客的地址的一致性

打开_config.yml

把 permalink 改为自己Ghost博客的地址格式

把 tag_dir: tags 改为 tag_dir: tag

测试

hexo generate
hexo server

打开http://localhost:4000/ 就可以看到自己的Hexo博客了!

部署到Github或Gitcafe

hexo deploy

高考过后准备入手一台android旗舰

恰逢一加手机发布了新品 于是我就买了

这次一加手机3和一加手机 X 一样采用了 AMOLED 屏幕材质，5.5寸1080p屏幕，效果只能算一般(P排的AMOLED PPI还要乘以0.82)，同时这个屏幕尺寸对于一些不喜欢“巨屏”的人来说是一个灾难，7.3mm 的厚度和金属后盖让手感更上一层楼。我拿出了我那部吃灰的MX 5 比较了一下，整机大小和MX5无异。

这一代和上一代一加手机2一样，采用了USB Type-C接口，不但在输入输出功率上比上一代提高并且支持正反插，这才是USB的未来啊！

然而很遗憾，一加手机3的USB Type-C并不是USB 3.0的标准 还是上一代一样采用USB 2.0

亮点

  • 支持支付宝和微信指纹支付 (国产Android手机 华为都支持 小米目前都不支持 魅族仅支持支付宝）

  • 6GB RAM 一加三是目前使用高通820的手机中 内存最大的手机

总结&评价

优点

  • 旗舰级配置 : 高通820+6GB RAM+64G ROM
  • 内核开源 : 官方开源内核 允许用户刷机
  • 全网通+双卡双待 : 电信用户的福音!!!终于可以一边用电信卡打电话 一边用联通卡上网了 (广东联通非常良心)
  • 非常容易抢 : 京东第一次抢购我花了3分钟就抢到了 闲鱼还有一堆人以原价出掉自己在京东抢购到的一加手机3
  • 指纹识别快 : 一摸指纹识别sensor就解锁了 识别速度和 iPhone 6S 有的一比
  • 外媒口中的「旗舰杀手」 : 瘾科技 、The Verge 和 Android Authority 都给予一加手机3不错的评价

缺点

  • 电量造假 : 一加3的快充电量造假 (因此我们放弃了充电对比环节)
  • 老板爱吹牛逼 : 一加在摄像上的调教并没有发布会上刘作虎吹的那么牛逼 (喜欢拍照的用户加几百块钱买三星 Galaxy S7 真心话)
  • 太「氢」了 氢OS简陋程度和原生Android有的一比 说得难听点就是个好看的花瓶
  • 不兼容的快充 : 与众多采用了高通820手机不同，一加 3 采用了 OPPO 的VOOC的快充方案而不是广泛采用的高通Quick Charge，意味着市面上大多数的快充头&充电宝都不兼容一加3的所谓「DASH极速闪充技术」

总结

我懒得码字了 直接引用 瘾科技的评价 吧

总的来说，一加 3 在软、硬件层面都能算是一台称职的当代旗舰，而且在手感和软件部分更是直中小编所求。所以如果说是「2016 年的旗舰杀手」，一加 3 在这短短的动手玩中已经能给小编这样的感觉了，只是问鼎年度机皇，还是多需要一点时间来观察。但考虑到官方依旧采用低定价的策略来销售一加 3，如此性价比，已经大大增加了它对一般消费者的吸引力了。

Shadowsocks Plugin for WHMCS 魔改版

本插件修改于 小蒋博客

感谢 小蒋博客 和 古尔数据 的无私贡献!

我的Shadowsocks插件地址: Github

为什么要修改

  • 因为我不喜欢直接显示IP 而显示域名，不显示IP 岂不美哉?
  • 喜欢折腾
  • 装逼
  • 小蒋版本提供的 Shadowsocks Manyuser 并不支持CHACHA20(在已经安装Libsodium的情况下)

所需安装:

分四部曲:

  • 架设API(以Nginx为例)
  • 建设Shadowsocks 后端 (简略带过)
  • 安装并设置好WHMCS 模块（最重要的一点）
  • 测试

在 Nginx+PHP-FPM 服务器上安装API

本文以军哥的 LNMP 一键包 为例

这里的 API 要和你准备用来给 Shadowsocks Manyuser 用的数据库是同一个服务器，否则就没意义了。

条件允许,建议API和WHMCS 装在同一个 VPS最好。

1、把 API 目录下的 config 目录和 shadowsocksapi.php 放到公共目录，提供公网访问。

举个例子，如果你正在使用军哥的 LNMP 一键包，那么你默认就把这个API文件放到 /home/wwwroot/default

2、新建一个数据库，并且导入 shadowsocks.sql 到数据库，新建数据库账户时注意设置所有地址访问。

3、修改 config 目录下的 configuration.php，把刚刚新建的数据库账户密码填进去。

4、限制 UA 访问，这一步特别重要。

修改Nginx的站点配置，如果是军哥LNMP包 Nginx的那么配置文件就在 /usr/local/nginx/conf/vhost

在合适的地方加入如下内容（例如 root /home/wwwroot/default 的下面）：

root  /home/wwwroot/default;

if ($http_user_agent != "ThisIsUA"){
        return 444;
    }

这里把上面的“ThisIsUA”改为一段 HASH，尽可能的长，例如：

root  /home/wwwroot/default;

if ($http_user_agent != "thM95vqtyvT5d7UCUmDYkHrh"){
    return 444;
}

为了安全和便利，建议把API listen 到一个奇葩的端口。
如 

listen 80;

改为

listen 9527;

然后用crontab -e添加计划任务

0 0 1 * * php -q /home/wwwroot/default/cron.php

参考 Breakwa11写的安装ShadowsocksR Manyuser教程)

首先把笔记本电池取出 按几十次电源键放电 然后在用螺丝刀拆开笔记本的挡板

然后把压着显卡和CPU的风扇拆掉 记得把风扇电源接口拔掉 不然容易扯断（我就差点扯断笔记本散热器电源了…)

用螺丝刀把螺丝向逆时针选择来解开CPU锁，取出原来的CPU 把新的CPU按上去 注意左下角的金色缺口哦 确认CPU安装牢固后再反锁CPU锁 涂好硅脂（我没有拍涂硅脂的样子）

在装回去就大功告成了!
